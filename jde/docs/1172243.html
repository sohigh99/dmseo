Introduction

Before you take any action make sure all the information is fully understood including the consequences of the change to be made. And the relationship between Work Order Header/Detail is understood properly.

 

For example, work order information can be stored as below,

  • F4801 Work Order Master File [header]
  • F3111 Work Order Parts List [detail]
  • F3112 Work Order Routing [detail]
  • F3114 Rate Schedule [detail]

: This document shall explain how to populate user reserved columns in F4801 but shall not cover how to populate it in detail tables listed above from this document.

 

 

Note: X4801 (Work Order Entry Master Business Function) is working very much different from other routine. One of reason is that the form W48013A is a Fix/Inspect form and Edit Line routine gets called only once in clicking OK button to validate input value.

 

 

P48013 Manufacturing Work Order Processing handles data as below,

 

  • X4801 - F4801BeginDocument (F4801 Begin Document): This can be called per session to initialize cache and get job number which makes cache(s) unique
  • X4801EditLine (F4801 Edit Line): Validate/defaults input value and call B4800210 X4801MfgEditLineModule (X4801 Manufacturing Editing Module)
    • B4800210 X4801MfgEditLineModule (X4801 Manufacturing Editing Module): Validate/manipulate input value handed by X4801EditLine and return updated cache to calling routine
  • X4801 F4801EndDocument: this function is to be called to commit cache created by Edit Line routine
    • B4800210 X4801MfgEndDocModule (X4801 Manufacturing End Doc Module. . Depends on settings, this routine can create additional routine (e.g., Co/By, Part list and so on)
  • X4801 F4801ClearWorkFile (F4801 Clear Work File): This routine can be called inside End Doc routine when all the transaction is ended successfully or when caches are to be terminated by cancellation and so on this can be called.
    • B4800210 X4801MfgTerminateModule (X4801 Manufacturing Terminate Module)

 

 

Before Start

  • Back up all the objects that may need to be changed
  • Make clear documentation on changes made Before and After
  • If there is any workaround for daily business, implement workaround rather than add additional data in User Reserved Columns

 

  1. Check business View in Form of W48013A
  2. Check Data Structure or Work File whether it contains User Reserved Columns from W4801, X4801 (in X4801.h and B4800210.h)
  3. Check whether B4800210 contains logic to handle User Reserved

 

  • Set proper action plan with object to be changed and object to be maintained

User Interface

User may enter User Reserved Columns in the Form of W48013A then this information has to be delivered to WO Header Master Business Function X4801EditLine to create WO Header Cache.

This document is to explain how we can implement this.

Action Plan

    • Verify whether existing Work File or data structure can hold User Reserved Columns for Work Order Header Cache Check X4801.h
    • Verify that exiting Business View for Work Order Header File have user reserved Columns Go to Business View (V4801C) Design and check it
    • Determine whether Data Structure or Business Function has to be changed or not Determine Data Structure which is used by X4801 (X4801EditLine) and B4800210 (X4801MfgEditLineModule) which is to be called in clicking OK button in W48013A
    • Verify whether existing routine handles User Reserved Columns. So through Edit Line cache has to be stored and End Doc routine commit cache to table F4801.
    • Implement it

 

Important:

    • The Work Order Header Master Business Function is unique in handling cache and it makes use of multiple caches because this table needs to store multiple transactions (SALES_PROCESS, REAL_PROCESS, MFG_PROCESS, CFG_PROCESS, MPM_CFG_PROCESS, REPETITIVE_PROCESS, MPM_PROCESS, CSMS_PROCESS, WARC_PROCESS, SUPR_PROCESS, EQUIPMENT_PROCESS).
    • All the cache has to be created by X4801 (X4801EditLine) and this cache has to be manipulated/saved in B4800210 (X4801MfgEditLineModule)
    • There are 3 different major caches in this routine,

DSX4801INTERNAL dsInternal; - Pointer to Internal Data structure

DSX4801CACHE dsCache;

DSX4801WORKCACHE dsWorkCache; - Pointer to Work Cache datastructure

lpdsF4801 - Pointer to input datatstructure F4801

: So this example show how to populate lpdsF4801 through Edit Line routine which is to be utilized through End Doc routine

Possible Implementation

  1. Work Order Revision form (W48013A) is written via Fix/Inspect form and data is to be handled by the business view V4801C (Manufacturing Work Order Header). Currently User Reserved Columns are not a member of data structure (or business view columns)

    • User Reserved Columns are not members of BSVW V4801C
    • So there has to be routine during Add/Update Mode
    • To handle BC value select User Reserved Columns and generate BSVW again (or re-create Typedef for BSVW V4801C)

 

Or just implement this using Data Dictionary Columns

 

  1. Verify that existing (Fix/Inspect) Form of W48013A [Work Order Details] does not contain User Reserved Columns

Now User Reserved Columns are ready to use because we have added User Reserved Columns into the associated Business View V4801C

User Reserved Columns are added into W48013A form

static void IB4800210_LoadX4801InputToCache(LPDSB4800210INTERNAL lpdsInternal,

LPDSX4801CACHE lpdsCache,

LPDSDX4801E lpdsInput)

{

*** Break In Code ***

MathCopy(&lpdsCache ->mnUserReservedAmount, &lpdsInput->mnUserReservedAmount);

MathCopy(&lpdsCache ->mnUserReservedNumber, &lpdsInput->mnUserReservedNumber);

jdeStrncpy((JCHAR *)(lpdsCache->szUserReservedCode),

(const JCHAR *)(lpdsInput->szUserReservedCode),

DIM(lpdsCache ->szUserReservedCode));

jdeStrncpy((JCHAR *)(lpdsCache->szUserReservedReference),

(const JCHAR *)(lpdsInput->szUserReservedReference),

DIM(lpdsCache ->szUserReservedReference));

memcpy((void *)(&lpdsCache->jdUserReservedDate),

(const void *)(&lpdsInput->jdUserReservedDate),

sizeof(JDEDATE));

*** Break In Code ***

}

 

n  Go to IB4800210_LoadCacheToF4801: This function copy the fields saved in the Main Cache back to the Work Order Header table (F4801).

static void IB4800210_LoadCacheToF4801(LPF4801 lpdsF4801,

LPDSX4801CACHE lpdsCache)

{

*** Break In Code ***

jdeStrncpy((JCHAR *)(lpdsF4801->waurcd),

(const JCHAR *)(lpdsCache->szUserReservedCode),

DIM(lpdsF4801->waurcd)-1);

MathCopy(&lpdsF4801 -> waurat, &lpdsCache->mnUserReservedAmount);

MathCopy(&lpdsF4801 -> waurab, &lpdsCache->mnUserReservedNumber);

jdeStrncpy((JCHAR *)(lpdsF4801-> waurrf),

(const JCHAR *)(lpdsCache ->szUserReservedReference),

DIM(lpdsF4801 -> waurrf));

memcpy((void *)(&lpdsF4801->waurdt),

(const void *)(&lpdsCache->jdUserReservedDate),

sizeof(JDEDATE));

*** Break In Code ***

}
